## 三、测试方案

### 3.1 API 压力测试 — Locust

**工具**: [Locust](https://locust.io/)

```python
# locustfile.py — 核心测试脚本框架

from locust import HttpUser, task, between

class DisasterPlatformUser(HttpUser):
    wait_time = between(1, 3)

    def on_start(self):
        # 登录获取 token
        resp = self.client.post("/api/auth/login", json={
            "username": "admin", "password": "admin123"
        })
        self.token = resp.json()["access_token"]
        self.headers = {"Authorization": f"Bearer {self.token}"}

    @task(3)  # 权重 3 — 高频操作
    def list_incidents(self):
        self.client.get("/api/incidents", headers=self.headers)

    @task(2)  # 权重 2
    def get_statistics(self):
        self.client.get("/api/statistics", headers=self.headers)

    @task(2)
    def list_resources(self):
        self.client.get("/api/resources", headers=self.headers)

    @task(1)  # 权重 1 — 低频操作
    def list_plans(self):
        self.client.get("/api/plans", headers=self.headers)

    @task(1)
    def health_check(self):
        self.client.get("/api/health")
```

**运行命令**:
```bash
pip install locust
cd backend
locust -f locustfile.py --host=http://127.0.0.1:8000 --users=50 --spawn-rate=5 --run-time=2m
```

**测试场景**:
| 场景 | 用户数 | 持续时间 | 验证目标 |
|------|:--:|:--:|------|
| 正常负载 | 20 | 5min | 全部指标达标 |
| 峰值负载 | 50 | 3min | 成功率 ≥ 99% |
| 持续负载 | 30 | 10min | 无内存泄漏、响应不退化 |
| 突发流量 | 50 spike | 2min | 系统自恢复能力 |

### 3.2 数据库查询性能

**工具**: PostgreSQL `EXPLAIN ANALYZE`

```sql
-- 关键查询性能分析
EXPLAIN ANALYZE SELECT * FROM incidents ORDER BY created_at DESC LIMIT 50;
EXPLAIN ANALYZE SELECT * FROM incidents WHERE status = 'pending_review' LIMIT 100;
EXPLAIN ANALYZE SELECT * FROM collected_events WHERE source = 'USGS' ORDER BY collected_at DESC LIMIT 20;
EXPLAIN ANALYZE SELECT category, COUNT(*) FROM incidents GROUP BY category;
```

**关注**:
- 全表扫描 (`Seq Scan`) → 需检查索引是否生效
- 大表 JOIN 性能
- `collected_events` 表随采集积累的增长趋势（预估每月 ~50MB）

## 四、系统架构性能说明

### 4.1 数据库连接池

```
pool_size = 20      # 核心连接数
max_overflow = 10   # 最大溢出连接
total = 30          # 峰值 30 连接
```

对 50 并发用户场景，每个请求平均持有时长 < 100ms，20 个核心连接足够支撑。

### 4.2 异步 I/O

FastAPI 基于 `uvicorn` + `asyncio`，所有数据库查询使用 `asyncpg` 异步驱动，I/O 不阻塞 worker 线程。

### 4.3 采集容错

- 5 分钟定时采集，历史失败不影响下轮
- 并行采集 USGS / GDACS / 天气 / 预警四个源
- 单源失败不阻塞其他源

### 4.4 AI 降级机制

```
DeepSeek API (8-25s)
    ↓ 超时/异常
本地模板引擎 (< 1s)
```
